# HWFrontQML

Qt Quick (QML + C++) приложение для мониторинга аппаратных метрик устройства (CPU/RAM/GPU и др.) с поддержкой подключения удаленного хоста и кастомизации виджетов на экране.

## Что делает проект

- Показывает экран подключения/авторизации устройства и экран с метриками.
- Переключает доступность страницы метрик в зависимости от состояния подключения.
- Получает данные через `HWConnector` и строит модель устройства через `DeviceBuilder`.
- Отображает метрики в карточках (`MetricCard`) с настраиваемым типом визуализации.
- Позволяет менять компоновку карточек через `WidgetLayoutDialog`.
- Содержит Android-конфигурацию для сборки мобильного пакета.

## Технологии

- **Qt 6 / Qt Quick Controls 2**
- **QML** для UI
- **C++17** для интеграции с backend-компонентами
- Внешние библиотеки/модули:
  - `HWConnector`
  - `DeviceBuilder`

## Структура проекта

- `main.cpp` — точка входа, регистрация контекста и загрузка `main.qml`.
- `core.h/.cpp` — мост между UI и backend (запросы, сигнал `deviceCreated`, доступ к `DesktopDevice`).
- `main.qml` — корневое окно и навигация между страницами.
- `PageAuthForm.qml` — экран подключения устройства.
- `PageDevicesInfo.qml` — экран мониторинга и виджеты метрик.
- `qml.qrc` — список QML-ресурсов, включаемых в бинарник.
- `android/` — Android manifest и gradle-конфигурация.

## Требования

1. **Qt 6.x** с модулями:
   - `Qt Quick`
   - `Qt Quick Controls 2`
2. Компилятор с поддержкой **C++17**.
3. Доступ к зависимостям (ожидаются по относительным путям):
   - `../HWConnector`
   - `../DeviceBuilder/src`
   - библиотеки в `../build/lib/` (Desktop/Android).

> Важно: без внешних компонентов `HWConnector` и `DeviceBuilder` проект не слинкуется.

## Быстрый старт (Desktop)

### Вариант 1: Qt Creator

1. Откройте `HWFrontQML.pro` в Qt Creator.
2. Выберите kit (Desktop Qt 6).
3. Запустите `Build` → `Run`.

### Вариант 2: консоль

```bash
qmake HWFrontQML.pro
make -j$(nproc)
./HWFrontQML
```

> На Windows вместо `make` используйте `jom`/`nmake` в зависимости от выбранного toolchain.

## Сборка Android

1. Настройте Android kit в Qt Creator (SDK/NDK/JDK).
2. Убедитесь, что используются файлы из каталога `android/`.
3. Соберите проект в режиме Android (`armeabi-v7a` указан в `.pro`).

## Поведение приложения

- После старта отображается страница подключения (`PageAuthForm`).
- При успешном создании устройства (`core.deviceCreated`) становится доступна страница с метриками.
- На странице метрик создается базовый набор виджетов (CPU/RAM/GPU), затем данные обновляются из модели устройства.
- Через заголовок экрана можно открыть настройки и диалог изменения компоновки виджетов.
